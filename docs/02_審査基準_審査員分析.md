# 審査基準・審査員分析

出典: https://challenge2026.odpt.org/ja/outline.html（2026-07-02取得）

## 公式の審査基準

特に以下の観点を重視し、総合的に評価される。

1. **社会課題解決への寄与**
2. **オープンデータ活用におけるインパクト**
3. **技術的な完成度**
4. **UI/UX面の完成度**

審査プロセス: 書類審査 → 一次審査（オンラインヒアリング、2027/1/23-24）→ 審査会 → 表彰式でプレゼン。

## 過去の講評から読み取れる評価傾向

- **交通弱者（視覚障碍者・車椅子・高齢者）支援は一貫して高評価**（2018最優秀「わたしのバス」、2017優秀「Suiっ都くん」等。坂村会長が講評で「オープンデータの重要な役割」と明言）
- **その年に新公開されたデータを即座に使う作品が評価される**（2018の駅構内図→Tokyo Station 3D Viewer、2025のデマンド交通→Mobiviz、規制データ→Safe Pedal）
- 「事業者では思いつかない」**発想の新規性**と、**実際に動く完成度**の両立が求められる
- 単機能でも「分かりやすさ」を突き詰めた作品が入賞する一方、最優秀は社会性＋技術＋完成度を兼備
- 自治体の政策立案支援（EBPM）系ツールが近年増加し、入賞している（コミバスをつくろう！、Mobiviz、Traffic Echo、Annelida）

## 2026年で狙うべき方向性（審査基準への当てはめ）

- 社会課題: 公式が明示する「少子高齢化・交通空白・オーバーツーリズム・バリアフリー」に直結させる
- データインパクト: 今年の目玉である**デマンド交通(GTFS-Flex)・地方都市圏データ・駅構内図・ほこナビ**を主役に使う。複数ソースのマッシュアップ（PLATEAU×気象×交通等）は加点要素
- 技術: リアルタイム（GTFS-RT）処理、3D/デジタルツイン、AI/機械学習は過去入賞作の定番強化軸
- UI/UX: 表彰式プレゼン・動画で伝わるビジュアルの分かりやすさ。「スライダー1つ」「かざすだけ」など操作の割り切りが好まれる
