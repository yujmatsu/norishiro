# 合体案「ノリシロ」: GTFS-Flex経路検索 × Transit MCP（低コスト構成）

方針: **「頭脳はアルゴリズム、LLMはただの入り口」**。
経路計算・予約締切判定・到達圏算出はすべて決定的アルゴリズムで実装し、LLMは持たない。AI対話はMCP経由で**利用者自身のLLM**（Claude / ChatGPT等）に任せる。

## なぜLLMコストがほぼゼロになるか

1. **MCPサーバーはLLMを呼ばない**。ただのAPIアダプタなので、推論コストは接続してきたユーザーのClaude/ChatGPTサブスク側で発生する（＝開発者負担0円）
2. コア機能（Flex対応ルーティング）は純粋なグラフ探索。LLM不要
3. アプリ内の「コンシェルジュ」は以下で代替:
   - ガイド型フロー（目的地・日時・制約をボタンとフォームで質問）→ 高齢者にはチャットより分かりやすい
   - 駅名・停留所のあいまい検索＋時刻表現のルールベース解析（LLM以前のNLU手法で十分）
   - 音声入力はブラウザ標準の **Web Speech API**（無料・端末内処理）
4. 審査用デモ動画でリッチな対話を見せたい場合のみ、無料枠（Google AI Studio等）か数百円分のAPIクレジットで収録

## アーキテクチャ（3層）

```
[データ層]  GitHub Actions（無料）で毎晩取得・変換
  ODPT GTFS-Flex / GTFSリポジトリ / GTFS-RT / GBFS / e-Stat施設
        ↓ 事前計算（グラフ構築・タイル化）
[エンジン層]  GTFS-Flex対応ルーティング
  案A: 自前RAPTOR実装(Rust/Go→WASM) … クライアントサイド実行=サーバー費0円
  案B: OpenTripPlanner 2(Flex v2部分対応・要検証) … Oracle Always Free等で稼働
        ↓ 同一コアを2つの口から公開
[インターフェース層]
  ① Webアプリ（MapLibre＋国土地理院タイル、ガイド型UI＋音声）
  ② MCPサーバー ＋ OpenAPI仕様公開 … 世界中のLLMエージェントが接続
```

- **スコープは全国**: 固定路線（GTFSリポジトリの全国バス・鉄道・フェリー＋ODPT鉄道・航空・GBFS）は全国分を取り込み、GTFS-Flexは提供のある自治体だけオーバーレイする。「データが無い地域は無いなりに表示」→ 空白が見えること自体が交通空白・オープンデータ未整備の可視化になり、作品の主張と一貫する
- 全国×0円の両立は**地域シャーディング**で解決: 都道府県/地域単位でグラフを分割ビルドし、検索地点に応じて数MBのシャードだけクライアントに配信（案A/WASM）。広域移動は鉄道・航空のみの軽量バックボーングラフで接続。サーバー側で持つ場合もOracle Always Free（RAM 24GB）で全国グラフが収まる見込み
- 案Aなら「ブラウザだけで動くオフライン対応Flexルーター」という技術的な見せ場にもなる

## 審査でのアピール構成

- **社会課題**: 交通空白（公式一丁目一番地）。「乗換アプリに出てこない移動手段を見えるようにする」
- **データインパクト**: GTFS-Flexの利用者向け消費は世界的にも希少 → MobilityData審査員に「標準策定への還元」として報告できる（実装で見つけた仕様課題のフィードバック）。MCP＋OpenAPI公開はPostmanエバンジェリスト審査員に直球
- **技術**: Flex対応ルーティングエンジン（＋WASM化なら加点）。LLMなしでAIエコシステムに接続する設計思想自体が話せるネタ
- **UI/UX**: 「予約締切まであと◯時間」の逆算表示、電話予約の心理障壁を下げる案内文生成（テンプレート）、高齢者向けガイド型UI
- **持続性**: 運用費ほぼ0円 → 「コンテスト後も維持できる」ことを明言できる（受賞後に死ぬ作品との差別化）

## リスクと対策

- **GTFS-Flexデータの品質・量が未知** → まず瑞穂町データで検分、エントリー後に9自治体分を検分。薄ければ「Flexが無い地域は電話予約情報を構造化して補完」する編集方針に切替
- **チャレンジ限定データは期間終了後に非公開化される** → サービス継続性の説明では「一般公開の瑞穂町＋自治体への公開働きかけ」を語る。作品はデータ差し替え可能な設計にする（Flexフィードを設定で追加できる汎用実装）
- **OTP2のFlex対応が不十分な場合** → 自前エンジン（案A）へ。対象地域限定なら実装量は制御可能
- **MCPだけだと審査員が触れない可能性** → Web UIを主役に、MCPは「エコシステム貢献」として添える
